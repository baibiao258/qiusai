# Prediction Accuracy Patterns (2026-06-30重写: 71场RECOMMEND实测)

基于 predictions_log.csv 截至 2026-06-29 共 **71场 RECOMMEND 已结算** 的完整统计分析。
相比 6/18 初版(仅4场xgb)，这是有统计意义的全量数据（137场filled中71场RECOMMEND）。

---

## 一、总准确率: 56.3%

RECOMMEND总计: 71场, SPF正确: 40场 = 56.3%。
全部137场filled(含SKIP/WATCH): 正确约50%。

**基准**: 随机猜SPF为33.3%。模型比随机高出23个百分点，但低于可投注盈利线(需>52.4%±margin)。

核心问题不是平均准确率，而是**置信度校准失败** — 高置信区间反而准确率更低。

---

## 二、置信度分桶准确率（核心发现）

| 置信区间 | 场次 | 正确 | 准确率 | 评价 |
|----------|------|------|--------|------|
| <40% | 8 | 3 | 38% | 🔴 近骰子 |
| **40-55%** | **15** | **5** | **33%** | 🔴 弱信号盲区 |
| 55-65% | 4 | 2 | 50% | 🟡 小样本 |
| **65-75%** | **12** | **11** | **92%** | 🟢 **甜区** |
| **75-85%** | **30** | **19** | **63%** | ⚠️ 衰退段 |
| **>=85%** | **2** | **0** | **0%** | 🔴 危险区 |

### 关键结论

1. **65-75% 是唯一可靠的置信区间** — 92%准确率。这是XGB模型的甜区。凡此区间内+RECOMMEND，可信度高。
2. **75-85% 准确率开始下滑至63%** — 模型对"比较确定"的比赛开始犯错。分桶内63% vs 甜区92% = 差了29个百分点，说明模型**过度自信**。
3. **>=85% 就是陷阱** — 2场全错（葡萄牙85%→D，法国84%→D）。历史也有英格兰85%→D。这是结构性偏差：XGB在极端高置信时忽略了平局风险。
4. **40-55% 是盲区** — 准确率仅33%，甚至低于随机。意味着模型对不确定的比赛给出错误的方向信号。
5. **<40% 和随机无区别** — 38%准确率，略好于瞎猜但不值得信任。

### 实用规则

```
当 max(H,D,A) 在 65-75% → 直接信任 RECOMMEND 方向
当 max(H,D,A) 在 75-85% → 信任但降级1级（留意冷门）
当 max(H,D,A) >=85%    → 高度警惕，反向考虑平局可能
当 max(H,D,A) 在 40-55% → 忽略 SPF 方向，关注其他信号
当 max(H,D,A) <40%     → 没信号，跳过
```

## 五、用户画像与行为模式（来自近期分析实践）

基于系统化数据分析的深审计方法固化：

### 分析入口
```
cd /root && python3 daily_jczq.py           # 先跑最新预测
python3 analyze_daily_results.py              # 规律推断
python3 analyze_recent.py                     # 准确率波动分析（自定义脚本）
```

### 分析管道标准流程
1. 检查 predictions_log.csv 总记录数和已结算数
2. 按 bet_action 分组统计分布
3. 按 date 分组计算每日SPF准确率
4. 分别统计全部场次和RECOMMEND场次的准确率
5. 按 model_route 分组对比
6. 按置信区间分桶计算准确率（关键）
7. 打印每个错误场次详情

### CSV字段关键信息
- `date`: 预测运行日（不是比赛日）
- `match_date`: 实际比赛日
- `bet_action`: 过滤标签（RECOMMEND / WATCH / SKIP_WORLD_CUP_FALLBACK）
- `actual_hda`: 实际赛果(H/D/A)
- `pred_h/d/a`: 胜平负预测概率（百分比值，66.0=66%）
- `pred_spf_pick`: SPF方向推荐
- `model_route`: 模型路由
- `actual_score`: 实际比分
- `kelly_pct`: Kelly仓位
- `goals_full`: 13档完整进球分布JSON

### 关键分析套路
1. **按置信分桶**是最高ROI的分析角度 — 直接揭示模型校准问题
2. **连续重复错误**比单场错误更值得关注 — 说明系统性盲区
3. **时间趋势**比总准确率更有用 — 分阶段看以排除初期不稳定期影响
4. **总准确率单值误导读者的程度很大**：71场56.3%看上去平庸，但近期26场73%说明模型已进入稳定性能期

## 六、已修复合正的系统性问题

### 6/17 初版指出的偏差（现均已解决或确认）

| 6/18判断 | 6/30确认 |
|----------|----------|
| Goals 预测模板化（永远说2） | ✅ 已修复 - 用 goals_full 13档取代score_full Top8 |

### 6/30 新发现的问题（待修复）

1. **置信度校准失败**: 最高置信区间(>=85%)准确率为0%，次级(75-85%)也仅63%
2. **40-55%盲区**: 模型最不确定时反而给出错误方向（33%准确率低于随机）
3. **重复错误不收敛**: 同场比赛在多天预测中持续错误（跨天预测不稳定是正常的，但方向应随机收敛）

## 七、比赛报告核心结论

analyze_daily_results.py 的9条规律引擎仍有效：
1. market_fallback=0% → 跳过
2. 甜区55-78%(强) | >80%(爆冷) | <40%(模糊)
3. SPF-Score方向一致: 必要非充分
4. Kelly>0加分，Kelly=0降1级
5. 演变轨迹: SPF方向改变→预警
6. 近tie检测: D/A或H/D概率差<5pp→不稳定
7. EV信号类型: 非SPF方向的高EV是价格信号非结果信号
8. Kelly归零: 演变中Kelly从>0→0是危险信号
9. 市场分歧: 模型≠市场时需警惕

**但需校正**: 规律2的甜区上限从78%修正为75%。>75%起准确率就开始下滑。
