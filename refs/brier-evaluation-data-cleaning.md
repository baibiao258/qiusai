# Brier 评估数据清洗 (2026-06-20)

## 问题

`predictions_log.csv` 的 Brier 评估存在两个数据污染源:

### 1. 0% 假预测 (去伪)
`market_fallback` 路由在 SPF 和 RQ 赔率全部为 0 时，`implied_probs_from_odds(0,0,0)` 因 `sum([0,0,0]) or 1.0 = 1.0` 返回 `{'H':0, 'D':0, 'A':0}`。写入 CSV 后产生 Brier=0.3333 的无效记录。

**影响 (2026-06-20 实测)**:
- 7/41 行 (17%) 是 0% 假预测
- 去除后 Brier: 0.2491 → 0.2318 (改善了 0.0173)
- Acc: 48.8% → 47.1% (说明假预测中有"碰对"的主胜)

### 2. 同队同日重复 (去重)
同一对球队在不同预测日期生成多条记录，但 match_date 相同（同一场比赛被多次预测）。
按 `(home_cn, away_cn, match_date)` 去重保留最新。

**注意**: 同一对球队在**不同 match_date** 的预测是独立的合法比赛（如 美国vs澳大利亚 在 6/17, 6/18, 6/19 各踢一场），不应视为重复。

## 路线细分 (CLEAN)

| 路由 | Brier | Acc | n | 判断 |
|------|-------|-----|---|------|
| xgb_dc_nat_11d | 0.2541 | 50.0% | 18 | 过自信: Acc更高但Brier更差 |

**关键洞察**: ML 模型准确率更高但因置信度过于极端，被 Brier 惩罚更重。这是 over-confidence 的典型症状。

## 修复记录

1. **`evaluate_brier.py`**: 新增 `clean()` 函数，所有评估模式默认通过 RAW→CLEAN 双版本对比展示。`--raw` 参数跳过清洗。
2. **`daily_jczq.py`**:
   - `fallback_market_predict()`: 无任何赔率时 return None
   - `record_prediction()`: pred_h/d/a 全 0 时 skip
   - `implied_probs_from_odds()`: 全零时返回均匀分布而非 [0,0,0]
