### Train-Serve Skew Diagnosis
30-dim model has 18 features with importance=0.0:
- form features (fh5_wr, fa5_wr, etc.)
- gold features (h2h_gd, tier_major, etc.)
- stage features (points_diff, rank_diff, etc.)

These features are computed differently during training (from snapshot) vs inference (from live data), causing noise that hurts performance.

### Recommendation
**Use nat model (11-dim) as primary for both pipelines.** Structural features (Elo, DC, market odds) are stable and reproducible. Soft features (form, gold, h2h) add more noise than signal due to train-serve skew.
