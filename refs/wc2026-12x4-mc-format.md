# 2026世界杯赛制变更与MC修正

## 背景

2026世界杯为48队参赛，赛制经过多次变更：

- **前期提案**：16组×3队（48/3=16），小组前2晋级R32
- **最终确认**：12组×4队（48/4=12），小组前2 + 8最佳第3晋级R32

## 赛制差异的影响

| 维度 | 16组×3队 (旧) | 12组×4队 (正确) |
|------|:-------------:|:---------------:|
| 小组赛场次 | 48 | 72 |
| 每队小组赛场次 | 2 | 3 |
| 晋级方式 | 小组前2 | 前2 + 8最佳第3 |
| 淘汰赛起始 | R32 | R32 |
| 轮次数 | R32→R16→QF→SF→F (4轮) | R32→R16→QF→SF→F (5轮) |
| 比赛总数 | 64 | 104 |

**影响**：
- 3队小组的最后一场可能出现"默契球"（第3场两队知道结果后可协商比分）
- 4队小组增加了比赛样本，降低了随机性
- 最佳第3名增加不确定性，不同组之间的积分/净胜球不可直接比较
- France在16×3赛制中被低估（仅有2场小组赛，冷门概率高）；12×4赛制中3场小组赛降低了冷门概率

## 修正时间线

1. 2026-05-21: Grok review指出MC赛制错误
2. 2026-05-21: 从The Odds API赛程验证确认为12组格式
3. 2026-05-21: 构建 `/root/data/2026_groups.json`（12组×4队，基于Matchday1赛程+Elo分配）
4. 2026-05-21: 重写 `_sim_worker` 函数（12组循环赛→最佳第3→R32→5轮KO）
5. 2026-05-21: 更新 `wc_2026_final.py` 加载分组文件

## 代码改动摘要

```python
# _sim_worker 函数签名变更:
# 旧: def _sim_worker(mc_cache_dict, elo, seed, n_sims, teams):
# 新: def _sim_worker(mc_cache_dict, elo, seed, n_sims, teams, groups):

# 小组赛循环变更:
# 旧: 3 pots × 16队 → 随机分16组 → 每组2场 → 前2晋级
# 新: 加载12个固定分组 → 每组6场 → 前2 + 8最佳第3 → 32强
```

## 分组数据

保存路径: `/root/data/2026_groups.json`

格式:
```json

  "A": ["Mexico", "South Africa", "Spain", "Iran"],
  "B": ["South Korea", "Czech Republic", "Argentina", "Algeria"],
```

## 淘汰赛签表变更（v2.0 修复版）

### Random Shuffle 模式（v1.x，已废弃）

旧版 `_sim_worker` 函数在 R32 签表使用 `_rnd.shuffle(qualifiers)` 随机配对。这导致：
- 强队路径被平滑，冠军概率被压低（Spain 从 20%+ 降至 13%）
- 中游队/东道主被错误抬高
- 同一队在不同运行间结果波动大

### Elo 排名配对（v2.0 当前）

已替换为 1v32 配对：按 24 支小组出线队 + 8 支最佳第三的 Elo 评分从高到低排序后 1v32、2v31……配对。

优点：
- 签表一致可复现
- 强队前半区优势保留
- 不会把小组第一配给另一个小组第一

局限：
- 不是 FIFA 官方 R32 对阵规则（待正式分组后实现）
- Elo 排序近似但不等于官方 POTS

## 东道主灵敏度（v2.0 新增）

管线自动用 `host_bonus=[0.0, 0.07, 0.1445]` 三组参数运行 50K 冠军模拟，输出灵敏度对照。

行为：
- `wc_2026_final.py` 自动输出灵敏度分析到日志
- `final_results.json` 的 `summary.host_bonus_info` 字段记录使用值
- 推荐冠军票前必须审阅灵敏度结果
