## P0#1: 路由决策显式日志

### 背景
`predict_match_wrapper()` 用 try/except 串联三个模型路由，异常吞没导致静默降级。当一个路由返回 None 时，没有通道知道"尝试了什么、为什么失败、选了什么"。

### 代码改动

**daily_jczq.py** (`predict_match_wrapper`):
```python
# 每个尝试记录 {'model': name, 'success': True/False}
tried = []
r = _try_club_predict(home, away)
tried.append({'model': 'club', 'success': r is not None})
if r is not None:
    source = 'club'
else:
    r = _try_hybrid_predict(home, away, league)
    tried.append({'model': 'intl', 'success': r is not None})
    if r is not None:
        source = 'intl'
    else:
        tried.append({'model': 'legacy', 'success': False})
        return None

r['source'] = source
r['routing'] = {'tried': tried, 'selected': source}
```

### 输出格式
```python
r['routing'] = {
    'tried': [
        {'model': 'club', 'success': True},
        {'model': 'intl', 'success': False},  # 仅当 club 失败时出现
    ],
    'selected': 'club'  # club | intl
}
```

### 消费方
`build_prediction_bundle()` 中的 `model_route` CSV 字段来源为 `p.get('model', 'unknown')`，与此 routing 日志无关。routing 日志用于终端调试/监控，不写入 CSV。

## P0#2: 500.com 熔断兜底

### 背景
`scrape_500_odds_today()` 的异步爬虫全量熔断时直接返回 `[]`，导致当日所有场次失去市场赔率参考。竞彩场次覆盖未来 1-8 天，历史快照中的赔率虽有延迟但优于无数据。

### 代码改动

**新增函数**: `_load_fallback_odds()` (daily_jczq.py)

**数据源**: `/root/data/odds_history.json` (由 `pre_match_odds_refresh.py` cron 每 30 分钟更新)

**过滤逻辑**:
1. 只取 `entry.get('start', '') >= today_str` 的场次
2. 转换赔率映射逻辑与 `scrape_500_odds_today()` 完全一致 (含 spf/nspf/rq 映射规则)
3. 标记 `std_odds_source='fallback_stale'`

**熔断分支改动**: 原 `return []` 改为:
```python
fallback = _load_fallback_odds()
if fallback:
    print(f'🟡 500.com 熔断, 使用 odds_history.json 兜底 ({len(fallback)} 场, 标记 stale)')
    return fallback
print(f'🔴 500.com 全量熔断, 无兜底数据, 跳过市场校准')
return []
```

### 验证
```bash

python3 -c "

    raw = json.load(f)

for e in valid[:3]:
    print(f'  {e.get(\"no\")} {e.get(\"home\")} vs {e.get(\"away\")} ({e.get(\"start\")})')
"
```

### 已知限制
- `odds_history.json` 不是完整快照 (当前 17 场, 覆盖 6/14-6/18)
- 赔率可能滞后 (cron 30 分钟间隔)
- `fallback_stale` 标记仅用于下游识别, EV 计算建议额外折扣
